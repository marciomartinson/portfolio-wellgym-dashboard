# WellGym — Dashboard de Validação de Persona

Página navegável com análise quantitativa simulada (n = 100) para validação e refinamento das protopersonas **Marina Castro** e **Henrique Mendes** — usuários de apps de treino e dieta no Brasil.

## O que tem neste dashboard

| Seção | Conteúdo |
|---|---|
| **Visão Geral** | KPIs, distribuição da amostra, gráficos de perfil, pontos de atenção |
| **Validação** | 12 hipóteses de dor testadas em escala 1–5, status de cada hipótese |
| **Funcionalidades** | Ranking Q19 (escolha de até 3 prioridades), apps usados, gap analysis |
| **Personas Finais** | Marina e Henrique refinados com dados reais, delta protopersona → persona validada |

## Principais achados

- **Histórico confiável** (49% na Q19) é a funcionalidade mais pedida — ausente no topo das protopersonas
- **Dores mais validadas**: pular/reordenar exercícios (68%), macros escondidos (67%), app lento (66%)
- **Hipótese descartada**: comunicação proativa (38%, menor score do estudo)
- **82%** dos respondentes pagariam por uma versão premium
- **Faixa de preço alvo**: R$15–R$30/mês

---

## Contexto do projeto

Pesquisa quantitativa com dados fictícios simulados, produzida como parte de um estudo de UX/Product Design para o **WellGym** — app conceito de treino e dieta com foco em performance e usabilidade real.

**Metodologia:** escala Likert 1–5 em 12 hipóteses de dor + priorização por escolha múltipla (Q19) + intenção de pagamento.
